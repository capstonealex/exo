C\+A\+Nopen\+Node is free and open source C\+A\+Nopen Stack.

C\+A\+Nopen is the internationally standardized (EN 50325-\/4) (\href{http://can-cia.org/standardization/technical-documents}{\texttt{ Ci\+A301}}) C\+A\+N-\/based higher-\/layer protocol for embedded control system. For more information on C\+A\+Nopen see \href{http://www.can-cia.org/}{\texttt{ http\+://www.\+can-\/cia.\+org/}}

C\+A\+Nopen\+Node is written in A\+N\+SI C in object-\/oriented way. It runs on different microcontrollers, as standalone application or with R\+T\+OS. Stack includes master functionalities. For Linux implementation with C\+A\+Nopen master functionalities see \href{https://github.com/CANopenNode/CANopenSocket}{\texttt{ https\+://github.\+com/\+C\+A\+Nopen\+Node/\+C\+A\+Nopen\+Socket}}.

Variables (communication, device, custom) are ordered in C\+A\+Nopen Object Dictionary and are accessible from both\+: C code and from C\+AN network.

C\+A\+Nopen\+Node homepage is \href{https://github.com/CANopenNode/CANopenNode}{\texttt{ https\+://github.\+com/\+C\+A\+Nopen\+Node/\+C\+A\+Nopen\+Node}}\hypertarget{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md0}{}\doxysection{C\+A\+Nopen Features}\label{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md0}

\begin{DoxyItemize}
\item N\+MT slave to start, stop, reset device. Simple N\+MT master.
\item Heartbeat producer/consumer error control.
\item P\+DO linking and dynamic mapping for fast exchange of process variables.
\item S\+DO expedited, segmented and block transfer for service access to all parameters.
\item S\+DO master.
\item Emergency message.
\item Sync producer/consumer.
\item Non-\/volatile storage.
\end{DoxyItemize}\hypertarget{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md1}{}\doxysection{Usage of the C\+A\+Nopen\+Node}\label{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md1}
C\+A\+Nopen\+Node itself doesn\textquotesingle{}t have complete working code for any microcontroller. It is only the library with the stack and drivers for different microcontrollers. It has example, which should compile on any system with template driver (drv\+Template), which actually doesn\textquotesingle{}t access C\+AN hardware. C\+A\+Nopen\+Node should be used as a git submodule included in a project with specific hardware and specific application.\hypertarget{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md2}{}\doxysection{Documentation, support and contributions}\label{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md2}
Code is documented in header files. Running \href{http://www.stack.nl/~dimitri/doxygen/index.html}{\texttt{ doxygen}} in project base folder will produce complete html documentation. Just open C\+A\+Nopen\+Node/doc/html/index.\+html in browser.

Report issues on \href{https://github.com/CANopenNode/CANopenNode/issues}{\texttt{ https\+://github.\+com/\+C\+A\+Nopen\+Node/\+C\+A\+Nopen\+Node/issues}}

Older and still active discussion group is on Sourceforge \href{http://sourceforge.net/p/canopennode/discussion/387151/}{\texttt{ http\+://sourceforge.\+net/p/canopennode/discussion/387151/}}

For some implementations of C\+A\+Nopen\+Node on real hardware see table below. \href{https://github.com/CANopenNode/CANopenSocket}{\texttt{ C\+A\+Nopen\+Socket}} is nice implementation for Linux devices. It includes command line interface for master access of the C\+A\+Nopen network. There is also some Getting started.

Contributions are welcome. Best way to contribute your code is to fork a project, modify it and then send a pull request.\hypertarget{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md3}{}\doxysection{Flowchart of a typical C\+A\+Nopen\+Node implementation}\label{md___users_williamcampbell__documents__git_hub_exo__a_l_e_x_src__c_a_nopen_node__r_e_a_d_m_e_autotoc_md3}
